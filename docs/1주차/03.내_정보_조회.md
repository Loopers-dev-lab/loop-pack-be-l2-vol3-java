## 요구사항
사용자는 사용자 정보를 조회할 수 있다(단, 자기 자신의 정보만 조회가 가능하다).
- 반환 정보: 로그인 ID, 이름, 생년월일, 이메일
- 이름은 마지막 글자를 마스킹해 반환 (마스킹 문자: `*`)
- 로그인 ID: `X-Loopers-LoginId`
- 로그인 PWD: `X-Loopers-LoginPw`

---

## 1단계: Layer별 책임 분리

#### Domain
- `User`: 이름 마스킹 로직 (`getMaskedName()`)

#### Service
- `UserService.getMyInfo(loginId)`: 사용자 정보 조회
  - 로그인 ID로 사용자 조회
  - 조회된 정보를 DTO로 변환하여 반환

#### Controller/DTO
- `UserController.getMyInfo()`: GET /api/v1/users/me
  - 헤더에서 `X-Loopers-LoginId` 추출
- `GetMyInfoResponse`: 로그인 ID, 마스킹된 이름, 생년월일, 이메일

## 2단계: 단위 테스트 (목록 추출 → 구현)

#### 테스트 목록
- [x] 이름 마스킹: 이름의 마지막 글자가 `*`로 마스킹된다
- [x] 이름 마스킹: 한 글자 이름은 `*`로 반환된다
- [x] 내 정보 조회 성공: 유효한 로그인 ID로 조회하면 사용자 정보가 반환된다
- [x] 내 정보 조회 실패: 존재하지 않는 로그인 ID로 조회하면 예외가 발생한다

## 3단계: 통합 테스트 (목록 추출 → 구현)

#### 테스트 목록
- [x] DB에 저장된 사용자 정보를 정상적으로 조회한다
- [x] 존재하지 않는 사용자 조회 시 예외가 발생한다

## 4단계: E2E 테스트 (목록 추출 → 구현)

#### 테스트 목록
- [x] GET /api/v1/users/me 요청 시 200 OK와 함께 마스킹된 이름이 포함된 사용자 정보가 반환된다
- [x] 존재하지 않는 로그인 ID로 요청 시 401 Unauthorized가 반환된다
